## 1. 為什麼這個程式碼庫符合申請資格？

這是一個從臨床排班情境衍生的醫療工作流程工具，用於產生多名醫師的月班表草稿，支援預假、指定醫師避排特定班次、列印與 CSV 匯出。專案目前由我持續維護，並正在補強 README、release、issue 與文件，讓它更適合長期開源維護。它的價值在於減少人工排班與反覆修表的時間成本。

## 2. 你會如何使用 API 積分來完成你的專案？

我會用 API credits 協助這個專案的維護工作，包括 pull request review、issue triage、README / 文件更新、release note / changelog 草稿產生、以及整理排班規則的測試案例與邊界情境。這類工作對小型但持續維護的工具特別有幫助，能讓我把更多時間放在臨床需求與排班規則修正上。

## 3. 還有其他想補充的資訊嗎？

這個工具不是要完全取代人工排班，而是先快速產生可用的候選班表，協助住院醫師、總醫師或科內行政在有限時間內完成初步安排，再進行人工微調。我目前會以保守、可驗證的方式公開專案資訊，不宣稱無法證實的使用量數字，並持續補齊 OSS 維護面需要的文件與流程。
