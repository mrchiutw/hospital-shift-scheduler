# Contributing

感謝你願意協助改善 Hospital Shift Scheduler。

## 專案定位

這個專案是給醫療排班工作流程使用的靜態前端工具。修改時請優先考慮：

- 臨床與行政使用者能否快速理解
- 排班限制是否表達清楚
- 輸出結果是否方便列印與人工微調
- 是否維持純前端、免安裝的使用門檻

## 開始之前

1. 先在 issue 描述你要修正的問題或新增的方向
2. 確認變更是否會影響：
   - 預假
   - 避排規則
   - 班次公平分配
   - 列印 / CSV 匯出
3. 所有新增或修改的文字請使用 UTF-8

## 專案結構

- `docs/`: 目前公開展示與 GitHub Pages 用的主版本
- `docs_legacy/`: 舊版展示頁，保留作比較或回溯
- `Reference/`: 歷史原型與早期輸出

## 本地測試

這個專案不需要建置流程。直接在瀏覽器開啟 `docs/index.html` 即可。

請至少手動確認以下情境：

1. 月份切換後天數會自動更新
2. 醫師人數可以在 3-80 之間調整
3. 可新增 / 移除避排規則
4. 預假 modal 可以點選與拖曳
5. 預假 modal 內按取消不應覆寫原本已儲存資料
6. 可成功產生班表
7. 可列印
8. 可匯出 CSV

## Release 與維護節奏

若這次變更會影響 README、demo 或 release 內容，建議一起檢查：

1. `README.md` 的功能描述、截圖與 demo 連結是否仍然正確
2. `docs/index.html` 是否能直接在瀏覽器中開啟並完成主要流程
3. `CHANGELOG.md` 是否需要補上這次的維護重點
4. 若要對外發布，再建立新的 GitHub release 或更新 release notes

若只做小修，保持 commit 範圍聚焦即可，不需要把 unrelated draft 或未使用資產一起混進去。

## Pull Request 建議

- 保持變更範圍聚焦
- 在 PR 描述中寫明使用情境
- 若改動 UI，請附上截圖
- 若改動排班邏輯，請補充測試步驟與預期結果

## 不建議的方向

- 未經需求確認就加入大量複雜規則
- 把工具改成需要後端或安裝依賴才可使用
- 在 README 或說明文件中加入無法證實的採用數字
